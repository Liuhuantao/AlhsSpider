# AlhsSpider

艾利浩斯图书馆小说爬虫，将 https://alhs.xyz 的文章批量下载为本地 txt 文件。

> 本项目由 Claude Code 编写。

## 特性

- **全站爬取** — 自动遍历所有列表页，收集文章链接
- **多线程并行** — 收集和下载阶段均使用线程池并行，`--workers` 控制线程数
- **全局速率控制** — 令牌桶限制 QPS，避免触发服务器防火墙
- **403 自动退避** — 检测到限速后指数降速，恢复正常后自动回升
- **智能过滤** — 跳过字数 < 500 的文章、分类为「互动小说」的文章
- **章节合并** — 自动检测多章节系列，跨范围下载全部章节合并为单个 txt
- **崩溃安全** — 系列章节逐章标记完成，中断后重跑不丢数据
- **断点续传** — 进度保存到 `progress.json`，中断后重跑不重复下载
- **页码范围** — 支持指定起始/结束页数
- **多格式兼容** — 自动适配标准 `<p>`、飞书 `<div.ace-line">`、旧版裸 `<div>` 三种段落格式

## 依赖

- Python 3.10+
- requests
- beautifulsoup4
- lxml

```bash
pip install requests beautifulsoup4 lxml
```

## 用法

```bash
# 全量爬取（从第 1 页到最后一页，默认 6 线程）
python main.py

# 指定线程数
python main.py --workers=10

# 指定页码范围
python main.py --start-page=5 --end-page=20

# 从某页开始，直到最后
python main.py --start-page=50

# 清除进度重新开始
python main.py --reset

# 测试：只扫描 5 页，下载 10 篇
python main.py --max-pages=5 --max-articles=10

# 只收集链接不下载（查看有多少文章）
python main.py --collect-only
```

### 参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--workers` | int | 6 | 并行线程数（收集+下载共用） |
| `--start-page` | int | 1 | 起始列表页 |
| `--end-page` | int | 0 | 结束列表页（0=直至最后） |
| `--max-pages` | int | 0 | 最多扫描页数（等同于 `--end-page`） |
| `--max-articles` | int | 0 | 最多下载篇数（测试用，0=不限制） |
| `--reset` | flag | — | 清除进度从头开始 |
| `--collect-only` | flag | — | 只收集链接不下载 |

### 线程数建议

| `--workers` | 效果 |
|---|---|
| 2-4 | 保守，几乎不会触发限速 |
| 5-8 | 推荐，速度与稳定兼顾 |
| 10-15 | 较快，偶发 403 会自动降速 |
| 15+ | 激进，容易触发长时间封禁 |

即使设置较大值，全局速率限制器也会将实际 QPS 控制在安全范围内。遇到 403 会自动退避，不会丢失数据。

## 过滤规则

文章在下载阶段被过滤（不是收集阶段），满足以下任一条件则跳过：

- 分类包含「互动小说」
- 标注字数 < 500
- 未标注字数且实际正文 < 500 字符

章节合并后的系列以**总字数**判断是否过滤。

## 反屏蔽机制

程序内置三层反屏蔽保护：

1. **全局速率限制** — 令牌桶算法，无论多少线程同时工作，总 QPS 不会超过上限（默认 4 QPS）
2. **403 自动退避** — 检测到 403 后指数增加等待时间（10s → 20s → 40s → … → 120s 上限），同时拉大基础间隔；成功后逐步恢复正常速度
3. **随机 UA 池** — 每次请求从 5 个真实浏览器 UA 中随机选取，附带 Referer 伪装

## 进度文件

`progress.json` 记录：

- `completed_urls` — 已处理的文章 URL（含已下载和已过滤）
- `collected_urls` — 已收集的全部链接
- `current_page` — 收集进度（当前扫描到的页码）
- `processed_series` — 已合并的系列 slug

删除此文件或使用 `--reset` 可重新开始。`novels/` 目录和 `progress.json` 已在 `.gitignore` 中排除。
